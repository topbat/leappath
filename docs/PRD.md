## 1. 产品概述

### 1.1 产品定位

求职助手是一款面向求职者的 **全生命周期 AI 辅助工具**，覆盖从简历准备、能力提升、面试演练、职位匹配到入职落地的完整求职链路。

### 1.2 目标用户

| 用户群体 | 核心诉求 |
|---------|---------|
| 应届毕业生 | 缺乏求职经验，需要全流程指导 |
| 1-3 年经验跳槽者 | 简历优化、薪资谈判、精准匹配 |
| 3-5 年经验进阶者 | 职业规划、深度准备、Offer 评估 |
| 跨行业转行者 | 技能差距分析、转行路径规划 |

### 1.3 产品形态

- **Web 应用**：功能完整的桌面端体验
- **微信小程序**：轻量级日常使用，依托微信生态

通过 `uni-app (Vue 3)` 实现一套代码双端运行。

### 1.4 核心价值主张

> **不止帮你找到工作，更帮你找到「对」的工作。**

## 3. 模块详细说明

### 3.1 简历中心 🏆 P0 最高优先级

#### 3.1.1 简历导入

| 项目 | 说明 |
|------|------|
| 支持格式 | PDF、Word (.doc/.docx)、图片 (JPG/PNG) |
| 解析方式 | OCR 识别 + NLP 结构化提取 |
| 解析内容 | 个人信息、教育经历、工作经历、项目经验、技能标签、证书等 |
| 异常处理 | 解析失败时引导手动填写，支持部分字段修正 |

**用户流程：**
```
上传文件 → AI 解析 → 预览结构化数据 → 确认/修正 → 入库
```

#### 3.1.2 简历创建

| 项目 | 说明 |
|------|------|
| 创建方式 | 引导式分步填写（个人信息 → 教育 → 经历 → 技能 → 其他） |
| 模板库 | 互联网 / 金融 / 咨询 / 快消 / 制造业 / 通用，各 ≥3 套模板 |
| 实时预览 | 填写同时右侧预览最终效果 |
| 草稿保存 | 自动保存，断点续填 |

#### 3.1.3 简历润色

| 项目 | 说明 |
|------|------|
| 润色范围 | 逐条优化工作经历、项目描述、自我评价 |
| 润色策略 | STAR 法则重写、量化成果（"提升了效率" → "效率提升 30%"） |
| 操作方式 | 逐条显示原文 vs 润色后，用户可选择采纳/修改/拒绝 |
| 风格选择 | 专业严谨 / 简洁有力 / 数据驱动 三种风格 |

**AI Prompt 约束：**
- 不编造数据，仅基于原文改写
- 保持真实性，标记不确定的量化数值供用户确认
- 匹配目标行业的关键词偏好

#### 3.1.4 简历评分

| 维度 | 权重 | 说明 |
|------|------|------|
| 内容完整度 | 25% | 各模块是否齐全，信息是否充分 |
| 关键词匹配 | 25% | 与目标行业/职位关键词的覆盖度 |
| 格式规范 | 15% | 排版、字体、间距、一页原则 |
| 竞争力 | 25% | 经历描述的深度、成果量化程度 |
| AIGC 检测 | 10% | 检测是否过度使用 AI 生成痕迹 |

**输出：** 总分 + 各维度分数 + 逐项改进建议（按优先级排序）

#### 3.1.5 定向优化

| 项目 | 说明 |
|------|------|
| 触发方式 | 输入目标职位 JD 链接或粘贴 JD 文本 |
| 优化内容 | 自动提取 JD 关键词，调整简历中的技能标签和经历侧重点 |
| 匹配度 | 展示优化前后的 JD 匹配度变化 |

#### 3.1.6 多版本管理

| 功能 | 说明 |
|------|------|
| 版本创建 | 基于主简历创建针对不同职位/公司的定制版本 |
| 版本命名 | 用户自定义名称（如"腾讯-后端开发"、"字节-产品经理"） |
| 版本对比 | 高亮显示不同版本之间的差异 |
| 默认版本 | 设置一个主版本作为投递默认简历 |

#### 3.1.7 简历导出

| 格式 | 说明 |
|------|------|
| PDF | 保持模板排版，支持 A4/Letter |
| Word | 可编辑格式，兼容 WPS/Office |
| 在线链接 | 生成可分享的在线简历页面（可选） |

### 3.2 模拟面试 P0

#### 3.2.1 面试类型

| 类型 | 描述 | 时长 |
|------|------|------|
| 技术面 | 算法、系统设计、项目深挖 | 30-60 min |
| 行为面 | STAR 行为问题、情景题 | 20-30 min |
| HR 面 | 薪资期望、职业规划、文化匹配 | 15-20 min |
| 群面模拟 | 无领导小组讨论（多人协作场景） | 30-40 min |
| 案例面 | 咨询/产品案例分析 | 30-45 min |

#### 3.2.2 行业方向

互联网、金融、咨询、快消、制造业、医疗、教育、新能源

#### 3.2.3 AI 面试官

| 能力 | 说明 |
|------|------|
| 动态追问 | 根据回答深度自动追问，模拟真实面试压力 |
| 岗位适配 | 基于目标公司 + 职位生成针对性问题 |
| 难度递进 | 从基础到深入，逐步提升难度 |
| 情绪感知 | 检测回答中的犹豫、不自信表达（文字维度） |

**对话示例：**
```
AI: 请介绍一下你在 XX 项目中扮演的角色和具体贡献。
用户: 我负责了后端 API 开发...
AI: 你提到了性能优化，具体做了哪些优化？QPS 从多少提升到多少？
用户: ...
AI: 如果让你重新设计这个系统，你会做哪些不同的决策？
```

#### 3.2.4 多模态交互

| 模式 | Web 端 | 小程序端 |
|------|--------|---------|
| 文字输入 | ✅ | ✅ |
| 语音输入 | ✅ | ✅（原生语音） |
| 语音合成 | 可选 TTS | 可选 TTS |

#### 3.2.5 实时反馈

每轮回答后即时展示：

| 反馈维度 | 说明 |
|---------|------|
| 内容质量 | 回答是否切题、深度如何 |
| 表达逻辑 | 结构是否清晰、是否有逻辑漏洞 |
| STAR 规范性 | 情境-任务-行动-结果 是否完整 |
| 改进建议 | 具体可操作的改进方向 |

#### 3.2.6 面试报告

面试结束后生成综合评估报告：

| 维度 | 分数 | 说明 |
|------|------|------|
| 沟通表达 | 1-10 | 语言组织、清晰度 |
| 逻辑思维 | 1-10 | 结构化思维、分析能力 |
| 专业深度 | 1-10 | 技术/行业理解深度 |
| 应变能力 | 1-10 | 面对追问的反应 |
| 综合评分 | 1-10 | 加权综合 |

**附加：** 优势总结、短板指出、针对性提升建议

#### 3.2.7 专项题库

| 分类维度 | 示例 |
|---------|------|
| 按岗位 | 前端、后端、算法、产品、设计、运营 |
| 按技术栈 | Java、Python、Go、React、Vue |
| 按难度 | 初级 / 中级 / 高级 |
| 按题型 | 算法题、系统设计、项目经验、行为题 |

#### 3.3.1 职位聚合

| 数据来源 | 方式 |
|---------|------|
| Boss 直聘 | API / 爬虫 |
| 猎聘 | API / 爬虫 |
| 拉勾 | API / 爬虫 |
| 企业官网 | 定向爬取 |
| 用户手动添加 | 粘贴 JD 文本 |

#### 3.3.2 智能匹配

| 项目 | 说明 |
|------|------|
| 匹配算法 | 简历画像 × JD 要求 → 多维度匹配度计算 |
| 匹配维度 | 技能匹配、经验匹配、学历匹配、薪资匹配、地点匹配 |
| 匹配度 | 百分比展示，≥80% 高匹配，60-79% 中匹配，<60% 低匹配 |

#### 3.3.3 差距分析

| 输出 | 说明 |
|------|------|
| ✅ 匹配点 | 已满足的 JD 要求（绿色标记） |
| ⚠️ 部分匹配 | 部分满足（黄色标记） |
| ❌ 差距点 | 不满足的要求（红色标记） |
| 💡 补齐建议 | 针对每个差距点的具体提升方案 |

#### 3.3.4 每日推荐

| 项目 | 说明 |
|------|------|
| 推荐频率 | 每日推送 Top 5-10 个职位 |
| 推荐依据 | 匹配度 + 用户偏好（行业、地点、薪资） |
| 去重 | 已投递职位不再推荐 |
| 推送渠道 | 小程序消息 / 邮件（可选） |

#### 3.3.5 投递追踪

看板式管理，支持拖拽流转：

```
┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
│  待投递   │→│  已投递   │→│  初筛中   │→│  面试中   │→│  已 Offer │
└──────────┘  └──────────┘  └──────────┘  └──────────┘  └──────────┘
                                              ↓
                                         ┌──────────┐
                                         │  已拒绝   │
                                         └──────────┘
```

每个卡片展示：公司名、职位、投递日期、最新状态、下次跟进时间

### 3.4 求职规划 P1

#### 3.4.1 能力自评

| 项目 | 说明 |
|------|------|
| 评估维度 | 硬技能（技术栈、工具）、软技能（沟通、领导力）、行业知识 |
| 评估方式 | 自评量表（1-5分）+ AI 辅助评估（基于简历和面试记录） |
| 可视化 | 雷达图展示能力画像 |

#### 3.4.2 技能差距分析

| 输入 | 目标职位/行业 |
|------|-------------|
| 输出 | 当前能力 vs 目标要求 的差距雷达图 |
| 差距分级 | 关键差距（必须补齐）/ 加分项（建议掌握）/ 锦上添花（可选） |

#### 3.4.3 学习路径

| 推荐类型 | 说明 |
|---------|------|
| 在线课程 | 慕课、极客时间、Coursera 等 |
| 书籍推荐 | 按技术栈和难度推荐 |
| 项目实战 | 开源项目、Side Project 建议 |
| 证书考取 | 行业相关认证 |

#### 3.4.4 求职时间线

倒推式规划：

```
目标入职日（2026-09-01）
  ↑
Offer 谈判期（8 月中旬）
  ↑
密集面试期（7 月 - 8 月）
  ↑
投递期（6 月下旬 - 7 月）
  ↑
准备期（现在 - 6 月下旬）
  ├── 简历优化
  ├── 技能提升
  └── 模拟面试
```

#### 3.4.5 每日任务

| 项目 | 说明 |
|------|------|
| 任务生成 | 基于时间线自动拆解每日任务 |
| 任务类型 | 投递 N 份简历、学习 X 小时、模拟面试 1 次 |
| 打卡追踪 | 完成打卡，连续打卡激励 |
| 进度统计 | 每周/每月完成率统计 |

#### 3.4.6 进度看板

| 展示内容 | 说明 |
|---------|------|
| 整体进度 | 百分比进度条 |
| 各模块完成度 | 简历 80%、技能 60%、面试练习 40% |
| 里程碑 | 已完成 / 即将到来 |
| 剩余天数 | 距目标入职日倒计时 |

| 起终点 | 选中房源 ↔ 公司地址 |
| 路线方案 | 展示多条路线（最短时间/最少换乘/最少步行） |
| 详细信息 | 总时长、换乘次数、步行距离、预估费用 |

#### 3.7.6 租金热力图

| 项目 | 说明 |
|------|------|
| 范围 | 城市/行政区/商圈 |
| 展示方式 | 热力图叠加在地图上，颜色深浅代表租金高低 |
| 交互 | 点击区域查看该区域均价 |
| 用途 | 快速识别高性价比区域 |

#### 3.7.7 周边配套

| POI 类型 | 展示方式 |
|---------|---------|
| 地铁站 | 步行距离 + 线路 |
| 商超 | 大型超市/便利店 |
| 餐饮 | 餐厅密度 |
| 医院 | 三甲/社区医院 |
| 健身房 | 距离和评价 |
| 银行 | ATM/网点 |

#### 3.7.8 预算计算器

| 输入 | 输出 |
|------|------|
| 税前月薪 | 税后收入估算 |
| 租金占比 | 建议 ≤30% 收入 |
| 推荐预算 | 租金上限金额 |
| 其他支出 | 水电网、通勤、餐饮预估 |

#### 3.7.9 多公司对比选址

| 场景 | 多个 Offer 对应不同办公地点 |
|------|---------------------------|
| 展示方式 | 地图上同时显示多个公司位置和通勤圈 |
| 对比维度 | 各公司周边租金均价、通勤便利度、生活配套 |
| 辅助决策 | 将租房成本纳入 Offer 总包对比 |

#### 3.7.10 城市横向对比

| 对比维度 | 说明 |
|---------|------|
| 薪资购买力 | 同岗位在不同城市的薪资水平 |
| 租金指数 | 各城市租金均价 |
| 生活成本 | 餐饮、交通、娱乐等综合成本 |
| 净结余估算 | 薪资 - 房租 - 基本生活开销 |
